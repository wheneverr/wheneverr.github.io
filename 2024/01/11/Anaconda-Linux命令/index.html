Anaconda Linux命令

基本命令

查看环境的基本信息

conda info

创建新环境

conda create -n [env_name] python=[version]

切换环境

conda activate [env_name]

更换环境为base

conda deactivate

删除指定环境

conda env remove –name [env_name]

查看当前环境下安装的包

conda list

安装指定包

conda install [package_name…]

卸载指定包

conda remove [package_name…]

根据requirement.txt安装环境

pip install -r requirements.txt

conda create –name your_env_name –file path/to/requirements.txt

安装torch2.0.1版本后运行代码报错

RuntimeError: GET was unable to find an engine to execute this computation

解决办法：

  1. 下载指定cuda版本的torch, torchvision, torchaudio。如：
# CUDA 11.8
pip install torch==2.0.1 torchvision==0.15.2 torchaudio==2.0.2 --index-url [https://download.pytorch.org/whl/cu118](https://download.pytorch.org/whl/cu118)
  1. 查看cuda版本 nvcc -V

如果与下载的torch版本不匹配，则

要解决这个问题，您可以在您的 .bashrc 或 .bash_profile 文件中临时修改 $PATH 和 $LD_LIBRARY_PATH，以确保首先访问 Conda 环境中的路径。这样做的话，当您激活 Conda 环境时，系统将优先使用 Conda 环境中的 CUDA 版本。

请按照以下步骤操作：

编辑 .bashrc 或 .bash_profile 文件：

打开您的 .bashrc 文件：

vim ~/.bashrc

添加环境变量修改命令：

在文件的末尾添加以下行：

# Temporarily remove system CUDA from PATH and LD_LIBRARY_PATH
export PATH=(echo "PATH" | sed -e 's|:/usr/local/cuda-11.1/bin||')
export LD_LIBRARY_PATH=(echo "LD_LIBRARY_PATH" | sed -e 's|:/usr/local/cuda-11.1/lib64||')`

这些命令会从 $PATH 和 $LD_LIBRARY_PATH 中移除系统 CUDA 的路径，从而确保在您的 Conda 环境中不会错误地使用它。

应用更改：

保存并关闭文件。然后在终端中运行以下命令以应用更改：

source ~/.bashrc

验证 CUDA 版本：

再次检查 $PATH 确保系统 CUDA 路径已被移除：

echo $PATH

然后激活您的 Conda 环境并检查 PyTorch 的 CUDA 版本：

`conda activate torch2.0.1python3.9

python -c “import torch; print(torch.version.cuda)”`

这样，当您在该 Conda 环境中工作时，应该不会再遇到由于 CUDA 版本冲突导致的问题。
